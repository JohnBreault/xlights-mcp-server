# Feature Spec: Creative Effect Sequencing

## Overview

The sequence engine should produce creative, human-like effect usage that looks intentionally choreographed — not algorithmically repetitive. A professional human sequencer builds energy, creates dramatic moments, uses call-and-response patterns, and ensures no two sections feel the same. This spec defines those behaviors.

## Current Limitations

1. **Static within sections** — the same chase runs for an entire 30-second section
2. **No intensity arc** — sections start and end at the same energy level
3. **No dramatic moments** — no blackouts, buildups, or surprise reveals
4. **No spatial choreography** — left/right models don't interact (call-and-response)
5. **Predictable palette use** — colors swap per section but don't evolve
6. **No silence awareness** — quiet gaps in the music aren't reflected in lighting

## Desired Behaviors

### 1. Intensity Arcs Within Sections
- **Verse**: start with bed only, add motion halfway through, subtle build
- **Chorus**: full energy from the start, optional peak accent at the end
- **Bridge**: dramatic shift — different palette, unique effects, slower movement
- **Intro**: gentle fade-in from darkness, gradually adding models
- **Outro**: reverse fade — models drop out one by one until just the tree/star remains

### 2. Dramatic Moments
- **Blackout before bass drop**: 0.5-1s of all lights off, then everything hits simultaneously
- **Flash-and-hold**: strobe on the beat, then freeze to a color wash
- **Build-and-release**: increasing twinkle density or chase speed leading into a chorus
- **Surprise color shift**: mid-section palette change on a key musical moment

### 3. Spatial Call-and-Response
- Left candy canes chase, then right candy canes chase (staggered by 1-2 beats)
- Arches cascade from center outward (arch 3 first, then 2+4, then 1+5)
- Tree effect triggers, then roofline responds, then ground elements follow
- During quieter moments, individual model groups solo while others hold a dim wash

### 4. Palette Evolution
- Palettes should transition across the song, not just switch per section
- Early sections: cooler tones. Peak sections: warm/bright. Ending: return to opening palette
- Within a section, allow 2-color → 3-color palette expansion as energy builds

### 5. Silence and Gap Awareness
- Musical breaks (>1s silence) → dim all to near-black or hold a gentle glow
- Brief gaps between phrases → quick fade-down, not sustained full brightness
- Re-entry after silence → timed to the first beat back, not the section boundary

### 6. Variety Guarantees
- No model group should have the same motion effect for more than 2 consecutive sections
- The accent effect type must vary between consecutive chorus sections
- At least 3 distinct visual "looks" across any song (palette + effect combination)

## Non-Goals

- Real-time reactive lighting (this is pre-rendered sequencing)
- Per-pixel custom animations (xLights handles that in its renderer)
- Manual timeline editing within this tool (that's what xLights UI is for)

## Success Criteria

A generated sequence for a 3-minute song should:
- Have visibly different looks for verse vs chorus vs bridge when viewed in xLights
- Include at least 1 dramatic moment (blackout, build, or flash) per song
- Show call-and-response between model groups at least twice
- Never have all models running identical effects simultaneously (except intentional flash moments)
- Look like it was sequenced by a human who listened to the song, not generated by an algorithm

## Affected Components

- `src/xlights_mcp/sequencer/engine.py` — primary implementation
- `src/xlights_mcp/xlights/effects.py` — may need new effect variants
- `src/xlights_mcp/audio/analyzer.py` — may need silence/gap detection
