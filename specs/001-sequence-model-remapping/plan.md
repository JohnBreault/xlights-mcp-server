# Implementation Plan: Sequence Model Remapping

**Branch**: `001-sequence-model-remapping` | **Date**: 2025-07-18 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-sequence-model-remapping/spec.md`

## Summary

Import an existing `.xsq` sequence or `.zip` package created for a different xLights show layout and intelligently remap/apply it to the user's own model layout. The system parses the imported sequence's XML, extracts model metadata from both sides, executes a multi-priority matching algorithm (exact name → word overlap → model type → prop similarity → pixel count fallback), generates a transparent mapping report, and writes a new `.xsq` file with all effects remapped to the user's models. Manual overrides allow users to correct any automatic decisions before generation.

## Technical Context

**Language/Version**: Python 3.11+ with type annotations throughout
**Primary Dependencies**: FastMCP (MCP protocol), Pydantic 2.x (data models), lxml 5.x (XML parsing/writing)
**Storage**: File-based — reads `.xsq` XML and `xlights_rgbeffects.xml`; writes new `.xsq` to show folder. Zip extraction to show folder for media assets.
**Testing**: pytest 8.x with pytest-asyncio 0.23+, ruff for linting
**Target Platform**: Cross-platform (macOS, Linux, Windows) — MCP server runs locally
**Project Type**: MCP server (library exposing tools via MCP protocol)
**Performance Goals**: ≤30 seconds for sequences with up to 500 models and 10,000 effect placements (SC-006)
**Constraints**: Read-only for user show config and imported files; new files only with incremented suffixes
**Scale/Scope**: Single-user local tool; sequences typically 10–500 models, 100–10,000 effects

**Structure Decision**: New `remapper/` package under `src/xlights_mcp/` follows the existing pattern where `xlights/` handles show I/O and `sequencer/` handles generation. This keeps the existing modules untouched and all new code in a focused package.
